# Security Policy

## Supported Versions

BarBop has not published a stable binary release yet. Security fixes currently
target the latest source on `develop` and will be included in the next release
candidate.

## Reporting a Vulnerability

Do not open a public issue containing exploit details, private notification
content, credentials, signing material, or personal system information.

Use GitHub's private vulnerability reporting for this repository. Include:

- the affected commit or app version;
- the macOS and hardware version;
- the impact and reproducible steps;
- whether Accessibility or notification permission is required;
- the smallest safe proof of concept, without private user data.

The maintainer will acknowledge a report as availability permits, validate the
issue, and coordinate a fix and disclosure. Routine crashes and non-sensitive
bugs can use the public bug report template.
